# QwenKB V1.0.1 技术文档

## 一、系统概述

QwenKB 是一个**本地私有知识库 RAG 系统**，核心能力：

1. 读取本地文档（PDF / Word / TXT / Markdown）
2. 用 Qwen3-Embedding 模型将文档向量化，存入 ChromaDB
3. 通过 MCP 协议将知识库暴露为 `search_knowledge_base` 工具
4. opencode / Claude Desktop 等 MCP 客户端可直接调用，检索文档内容

**一句话：把你的文档变成 AI 能搜索的数据库，不联网，无第三方平台依赖。**

## 三、核心概念术语表

| 术语 | 通用定义 | 本系统中的具体实例 |
|------|---------|-------------------|
| **Embedding（向量化）** | 把一段文字翻译成一串数字（向量），含义相近的文字向量也相近 | Qwen3-Embedding-4B 模型，通过 LM Studio 运行，将文本转为 2560 维向量 |
| **向量数据库** | 专门存储和检索向量的数据库，通过余弦相似度找最相近的记录 | ChromaDB，存储在 `chroma_db/` 文件夹中，嵌入式模式运行 |
| **RAG（检索增强生成）** | 先检索知识库找到相关内容，再把内容+问题发给 LLM 生成答案 | `search_knowledge_base` 工具实现检索，opencode 后端 LLM 实现生成 |
| **MCP（模型上下文协议）** | AI 工具连接外部数据源的统一协议，定义标准的工具发现和调用流程 | `mcp_server.py`，HTTP/8766，只暴露一个工具 |
| **MCP 工具** | MCP 服务器向外提供的一个可调用功能 | `search_knowledge_base(query: str) -> str` |
| **LLM（大语言模型）** | 能理解和生成自然语言的大模型 | DeepSeek API（opencode 后端），负责基于检索结果生成答案 |
| **文档切片** | 把长文档切成小块，每块独立向量化 | `split_text()` 函数，按中文标点+换行符分隔，每块 500 字，重叠 50 字 |
| **Windows 服务** | 开机后自动启动的后台程序 | nssm 注册的 `QwenKB-MCP` 服务，指向 `mcp_server.py` |
| **HTTP 端点** | 网络服务的访问地址 | `127.0.0.1:5000`（LM Studio）+ `127.0.0.1:8766`（MCP Server） |
| **Q4_K_M 量化** | 模型压缩方式，降低显存占用 | Qwen3-Embedding-4B-GGUF 的量化版本，2.3GB，1080Ti 可用 |

## 六、连接方式与协议详解

| 连接路径 | 协议 / 方式 | 方向 | 说明 |
|---------|------------|------|------|
| opencode → MCP Server | HTTP POST (MCP/JSON-RPC) | 客户端→服务器 | opencode 调用 MCP 工具时发送 JSON-RPC 请求 |
| MCP Server → LM Studio | HTTP POST (OpenAI API 兼容) | 客户端→服务器 | 发送文本获取 embedding 向量 |
| MCP Server → ChromaDB | 本地文件系统（嵌入式） | 直连 | 读取已存储的向量和文档 |
| build_kb.py → LM Studio | HTTP POST (OpenAI API 兼容) | 客户端→服务器 | 批量发送文本获取向量 |
| build_kb.py → ChromaDB | 本地文件系统（嵌入式） | 直连 | 写入向量和元数据 |

**注意：** MCP Server 不直接调用 DeepSeek API。DeepSeek 由 opencode 后端负责，MCP Server 只提供检索结果。

### 8.4 install_service.ps1 详解

#### 8.4.1 这个脚本是什么

`install_service.ps1` 是一个 **PowerShell 脚本**（`.ps1` 后缀，类比 Python 的 `.py`），它把 MCP 服务器注册为 Windows 服务，开机自动运行。

#### 8.4.2 nssm 是什么

**nssm = Non-Sucking Service Manager**，一个免费微型工具，只有一个 `nssm.exe` 文件（约 500KB）。

它的作用是充当「翻译官」：Windows 的 SCM（服务控制管理器）只能管理特定格式的程序，普通 `python xxx.py` 它不认识。nssm 把普通程序包装成 SCM 能理解的格式，注册进去。

| 类比 | 说明 |
|------|------|
| Windows SCM | 操作系统内置的「自动喂食器」——负责启动、停止、重启程序 |
| nssm | 把普通程序注册到喂食器里的「登记员」 |
| 你的 mcp_server.py | 被注册的普通程序 |

**你不需要手动下载 nssm。** 脚本第一次运行时，会自动检查 `nssm/` 文件夹下有没有 `nssm.exe`，没有就自动从 nssm.cc 下载解压。

#### 8.4.3 菜单功能说明

运行 `.\install_service.ps1` 后会看到 5 个选项：

| 输入 | 操作 | 内部执行了什么 | 使用场景 |
|------|------|---------------|---------|
| `1` | 安装 + 自动启动 | ①下载 nssm ②`nssm install` 注册服务 ③`nssm set` 设置属性 ④`nssm start` 立即启动 | **初次部署/重装** |
| `2` | 停止 + 卸载 | ①`nssm stop` 停止服务 ②`nssm remove` 从 SCM 注销 | **彻底移除服务**（不删文档和数据库） |
| `3` | 手动启动 | 只执行 `nssm start` | 之前手动停止了，想重新启动 |
| `4` | 手动停止 | 只执行 `nssm stop` | 临时关掉，不卸载 |
| `5` | 查看状态 | 只执行 `nssm status` | 看一眼服务是否在跑 |

菜单本质上是一个 `switch` 分支语句，让你用一个脚本管 5 件事，不用记 5 条不同命令。

#### 8.4.4 注册时做了什么

选择 `1` 后，脚本依次执行：

```powershell
nssm install QwenKB-MCP  D:\anaconda\envs\qwen-kb\python.exe  E:\桌面\qwen-embedding\mcp_server.py
nssm set    QwenKB-MCP  AppDirectory  E:\桌面\qwen-embedding
nssm set    QwenKB-MCP  DisplayName   "QwenKB MCP Server"
nssm set    QwenKB-MCP  Start         SERVICE_AUTO_START
nssm start  QwenKB-MCP
```

| 命令 | 含义 |
|------|------|
| `install` | "嗨 SCM，注册一个新服务，名叫 QwenKB-MCP" |
| `AppDirectory` | "工作目录是项目文件夹，保证相对路径（docs/chroma_db）能解析" |
| `DisplayName` | "在 services.msc 界面上显示这个名字" |
| `Start SERVICE_AUTO_START` | "每次开机自动启动，不等用户登录" |
| `start` | "现在立刻跑起来" |

#### 8.4.5 注册后如何确认成功

`Win+R` → 输入 `services.msc` → 回车 → 搜索 `QwenKB` → 状态应为「**正在运行**」，启动类型应为「**自动**」。

也可以浏览器访问 `http://127.0.0.1:8766/health`，返回 JSON 即正常。

#### 8.4.6 日常管理

注册之后，可以通过以下方式管理服务：

| 操作 | 方法 |
|------|------|
| 启动/停止/重启 | `services.msc` 中右键操作 |
| 改启动类型 | `services.msc` → 右键属性 → 启动类型 |
| 查看日志 | 打开 `E:\桌面\qwen-embedding\mcp_server.log`（滚动日志，保留最新 3 个备份） |
| 完全卸载 | 以管理员运行 `.\install_service.ps1` → 输入 `2` |

**卸载不会删除 `docs/` 文档和 `chroma_db/` 知识库**，只注销服务本身。
